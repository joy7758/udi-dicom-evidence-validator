# v0.2.0 Release Notes

v0.2.0 adds synthetic workflow traceability and optional FDO-style mapping
metadata while preserving the public validator boundary.

Included changes:

- Profile and schema support for `synthetic_workflow_trace_id`, `provenance`,
  and optional `fdo_mapping`.
- Receipt schema support for deterministic `trace_id` and provenance output.
- Public v0.2 synthetic examples and pass/fail fixtures.
- Validator checks for `trace_id_consistency` and `fdo_mapping_consistency`.
- CLI, API, demo, README, llms, AGENTS, and skill documentation updates.
- Tests for v0.1 and v0.2 compatibility.

Boundary: not clinical validation, not regulatory approval, not certification,
no PHI, offline fixture first, and live openFDA only by explicit opt-in.
