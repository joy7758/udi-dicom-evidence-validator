# Attachment Checklist

Use this checklist before sending recommendation materials to a department, collaborator, or reviewer.

## Safe Attachments

- `department-recommendation-report-cn.md`
- `executive-one-page-cn.md`
- `evidence-map.md`
- `boundary-and-risk-statement-cn.md`
- `hospital-it-brief-cn.md`
- `device-manufacturer-brief-cn.md`
- `medical-ai-data-brief-cn.md`
- Public repository URL
- Verified Zenodo DOI URL
- Paper DOI URL

## Exclude

- Any PHI.
- Any raw DICOM.
- Any private suite or private service material.
- Any real sample, local intake package, customer workflow, NDA material, or deployment artifact.
- Any claim of clinical validation, regulatory approval, certification, or hospital deployment.

## Required Footer

Boundary: no PHI, no raw DICOM, not clinical validation, not regulatory approval, not certification, public synthetic examples, Device UID != UDI-DI.
