# DOI Readiness v0.6

The repository is prepared for DOI-ready software archiving through GitHub
Release assets and metadata files. DOI-ready means the metadata can be reviewed
before Zenodo or another archive service assigns a DOI.

Rules:

- Do not claim a DOI before assignment.
- Do not put private suite or private service material into the public DOI
  record.
- Archive only public validator source, synthetic public examples,
  reproducibility material, and public release assets.
- Keep the boundary: no PHI, no raw DICOM, not clinical validation, not
  regulatory approval, and not certification.
