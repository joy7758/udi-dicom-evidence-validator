# v0.8.0 DOI Readiness

Release state: `DOI_ARCHIVE_READY_PENDING_ZENODO_ENABLEMENT`

v0.8 prepares DOI archive review and paper finalization support for the public
validator. It does not change validator behavior, schemas, public examples, API,
CLI, or receipt semantics.

Required checks:

```bash
python scripts/build_doi_archive_review_package.py
python scripts/check_doi_archive_review_package.py artifacts/doi-archive-review-v0.8
python scripts/check_no_fake_doi.py
python scripts/build_paper_finalization_bundle.py
python scripts/check_paper_finalization_bundle.py artifacts/paper-finalization-bundle-v0.8
python scripts/check_citation_metadata_consistency.py
pytest -q
ruff check .
mypy src
```

Release trigger is blocked until a human confirms Zenodo GitHub integration is
enabled for the public repository. The resulting GitHub Release should be
reported as `ZENODO_ARCHIVE_PENDING` until a real record URL and DOI are
returned.

Boundary: no PHI, no raw DICOM, not clinical validation, not regulatory
approval, not certification, no fake DOI, Device UID != UDI-DI, offline fixture
first, live openFDA explicit opt-in, FDO-style mapping only, and no robot
operation evidence.
