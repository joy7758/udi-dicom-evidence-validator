# Security Policy

This repository has a no-PHI policy. Do not add patient names, patient IDs,
accession numbers, dates of birth, addresses, phone numbers, emails, MRNs,
clinical reports, or original DICOM pixel data.

Report vulnerabilities through the repository issue tracker or a private
maintainer channel. Do not attach real medical samples to public issues.

Private conformance cases and customer-specific mapping rules are excluded from
this public repository. Dependency updates should preserve deterministic
validator output. Live registry lookup is optional and must not be used for
medical decision support.
