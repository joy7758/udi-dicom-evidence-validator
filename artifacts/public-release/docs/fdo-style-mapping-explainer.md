# FDO-Style Mapping Explainer

The v0.2 profile allows optional `fdo_mapping` metadata to describe how a
manifest, artifact, or registry fixture could be discussed using FAIR Digital
Object vocabulary.

Boundary:

- The fields are optional.
- The fields do not register persistent identifiers.
- The validator does not claim official FDO conformance.
- FDO-style mapping does not change clinical, regulatory, or certification
  status.
