# Known Limitations v0.4

- Public examples are synthetic only.
- The validator does not process PHI.
- The validator does not replace PACS or VNA.
- The validator does not provide clinical validation.
- The validator does not provide regulatory approval.
- The validator does not provide certification.
- Live openFDA access is not the default and remains explicit opt-in.
- FDO-style fields are metadata mapping aids, not an official FDO
  implementation claim.
- The public MCP skeleton exposes public helper functions only; it does not
  access private conformance cases or private service workflows.
